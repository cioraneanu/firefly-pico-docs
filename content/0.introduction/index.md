## 💡 About

For several years I've enjoyed tracking my expenses in proprietary mobile apps, but after getting into self-hosting, I wondered if there was an alternative that gives me the same functionality without
losing control over my data.

I came across **[Firefly III](https://github.com/firefly-iii/firefly-iii)** and instantly fell in love with its features. Since it offers a REST API, I decided to build a web app that feels mobile-native. I also wanted to ensure it passed the "
Wife Approval Certification"—making the process as short as possible while still using tags and categories for those beautiful reports.

Several [bad] ideas later, I've come to this version: **A transaction assistant** 🎩.

### The Assistant in Action

| Action                                            | Result                                                                                                                                                                                                  |
|:--------------------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Write "tag" + "amount" <br/> <br/> Ex. "elctr 28" | - Fuzzy search finds "electricity," sets your tag, category and description, <br/> - Sets the date to "now," <br/> - Sets Amount to 28<br/> - Sets your default asset account.<br/> <br/> Just hit "Save" |

## 🚀 Features

- ✅ Beautiful clean minimalistic UI 🔥 subtle animations and dark-theme support 🌛
- ✅ Optimized for mobile, with PWA support for a native feel 📱
- ✅ The Assistant makes recording an expense feel like magic ✨
- ✅ Expand Firefly data with icons for all resources 🎨
- ✅ Expand Firefly "tags" with the option of subtags for better granularity
- ✅ Add "templates" to make auto-completing fields a breeze 📢
- ✅ Beautiful dashboard making it easy to keep an eye on everything that matters 📈
- ✅ Lots of tweaks and settings 🔧
- ✅ Free and open-source
